LOCK 1 PROOF — PERIODIC POSITIVE-ENGINE / NEGATIVE-GHOST LOCK
=================================================================

Purpose
-------

This file contains:

1. A clean writeup of the Lock 1 proof.
2. The exact algebra behind the proof.
3. A standalone Python verifier that computes the affine block data, fixed
   point, action classification, and sample/exhaustive bounded checks.

Lock 1 statement
----------------

A positive odd Collatz orbit cannot be powered forever by a repeating
positive-action exponent word.

Equivalently:

    Any repeated block word w with net lift has its affine fixed point on the
    negative side. Therefore it is a negative ghost, not a positive integer
    cycle.

This kills positive periodic engines.


Odd-only Collatz block notation
-------------------------------

Let an exponent word be:

    w = (a_0, a_1, ..., a_{k-1})

where each a_i >= 1 and:

    a_i = v2(3*x_i + 1)

The odd-only Collatz step is:

    x_{i+1} = (3*x_i + 1) / 2^{a_i}

After applying the whole word w once, the block map has affine form:

    F_w(x) = (3^k*x + B_w) / 2^A

where:

    k = length(w)
    A = a_0 + a_1 + ... + a_{k-1}
    B_w > 0

The offset B_w is produced by the accumulated +1 terms.

The recurrence for B_w is exact:

    Start:
        A_0 = 0
        B_0 = 0

    Append exponent a_j:
        B_{j+1} = 3*B_j + 2^{A_j}
        A_{j+1} = A_j + a_j


Action classification
---------------------

The block multiplier is:

    3^k / 2^A

There are three cases.

1. Positive action / net lift:

       3^k > 2^A

2. Contractive action / net drop:

       3^k < 2^A

3. Neutral action:

       3^k = 2^A

Neutral action is impossible for nonempty words, because 3^k and 2^A are powers
of distinct primes. They can be equal only if:

    k = 0 and A = 0

which is the empty word, not a nontrivial Collatz block.


Fixed point of a repeated block
-------------------------------

A repeating block engine would require a fixed point:

    F_w(x) = x

Using the affine form:

    (3^k*x + B_w) / 2^A = x

Multiply by 2^A:

    3^k*x + B_w = 2^A*x

Move x terms together:

    B_w = (2^A - 3^k)*x

Therefore:

    x = B_w / (2^A - 3^k)


Lock 1 proof
------------

Assume w is a nonempty positive-action word.

Then:

    3^k > 2^A

so:

    2^A - 3^k < 0

Also:

    B_w > 0

because every nonempty block accumulates positive +1 offset.

Therefore the fixed point is:

    x = B_w / (2^A - 3^k) < 0

So the only affine fixed point of the positive-action block is negative.

That negative fixed point is the ghost:

    gamma_w = - B_w / (3^k - 2^A)

Since gamma_w < 0, no positive integer can be the fixed point of that repeated
positive-action block.

Even stronger, for x > 0:

    F_w(x) - x
      = ((3^k*x + B_w) / 2^A) - x
      = ((3^k - 2^A)*x + B_w) / 2^A

For positive action:

    3^k - 2^A > 0

and:

    B_w > 0

so for every x > 0:

    F_w(x) - x > 0

Thus repeated application of a positive-action word strictly increases every
positive input. It cannot return to its starting value and cannot form a
positive periodic engine.

Conclusion:

    Every positive-action repeating block has a negative ghost fixed point.
    Therefore no positive integer orbit can be powered forever by a repeating
    positive-action word.

This is Lock 1.


What this proof does and does not cover
---------------------------------------

Lock 1 covers repeated positive-action engines.

It does not need to handle contractive words. Contractive words belong to the
descent side of the framework.

It does not need to handle bounded nonperiodic gliding. That is Lock 3.

It does not need to handle unbounded reserve/corridor chaining. That is Lock 4.


Standalone Python verifier
--------------------------

Save the code below as:

    lock1_verifier.py

and run:

    python3 lock1_verifier.py

The script:
- computes A, k, B_w for a word;
- computes the affine fixed point;
- classifies the action;
- verifies that every positive-action word in a bounded enumeration has a
  negative fixed point;
- verifies that F_w(x) > x for positive sample x when the word has
  positive action;
- prints concrete examples and summary counts.

PYTHON CODE

from fractions import Fraction
from itertools import product


def affine_data(word):
    """
    Return (A, k, B) for exponent word w.

    F_w(x) = (3^k*x + B) / 2^A

    Recurrence:
        B_{j+1} = 3*B_j + 2^{A_j}
        A_{j+1} = A_j + a_j
    """
    A = 0
    B = 0
    k = 0

    for a in word:
        if a < 1:
            raise ValueError("All exponents must be positive integers.")
        B = 3 * B + (1 << A)
        A += a
        k += 1

    return A, k, B


def action(word):
    """
    Classify a word by comparing 3^k and 2^A.
    """
    A, k, B = affine_data(word)
    two_A = 1 << A
    three_k = 3 ** k

    if three_k > two_A:
        return "positive"
    if three_k < two_A:
        return "contractive"
    return "neutral"


def fixed_point(word):
    """
    Return the exact fixed point x = B / (2^A - 3^k)
    as a Fraction.
    """
    A, k, B = affine_data(word)
    return Fraction(B, (1 << A) - (3 ** k))


def F(word, x):
    """
    Apply the affine block map F_w to x exactly as a Fraction.
    This does not check whether x follows the word as a Collatz
    valuation path. Lock 1 is about the affine repeated engine.
    """
    A, k, B = affine_data(word)
    return Fraction((3 ** k) * x + B, 1 << A)


def verify_positive_engine(word, sample_x_values=(1, 3, 5, 7, 11, 101)):
    """
    Verify the Lock 1 claims for one positive-action word:
    - fixed point is negative;
    - F_w(x) > x for positive sample x.
    """
    A, k, B = affine_data(word)
    two_A = 1 << A
    three_k = 3 ** k

    assert B > 0, f"B should be positive for nonempty word {word}"

    if three_k <= two_A:
        return None

    fp = fixed_point(word)
    assert fp < 0, f"Positive-action word has nonnegative fixed point: {word}, fp={fp}"

    for x in sample_x_values:
        y = F(word, x)
        assert y > x, f"Positive-action word did not increase x: word={word}, x={x}, y={y}"

    return {
        "word": word,
        "A": A,
        "k": k,
        "B": B,
        "2^A": two_A,
        "3^k": three_k,
        "fixed_point": fp,
        "ghost": fp,
    }


def enumerate_words(max_len=6, max_a=6):
    """
    Enumerate exponent words of length 1..max_len
    with entries 1..max_a.
    """
    for length in range(1, max_len + 1):
        for word in product(range(1, max_a + 1), repeat=length):
            yield word


def run_examples():
    examples = [
        (1,),
        (1, 1),
        (1, 2),
        (2,),
        (2, 1),
        (1, 1, 1),
        (1, 2, 1),
        (1, 2, 2),
        (1, 1, 2, 3),
    ]

    print("Concrete examples")
    print("=================")

    for word in examples:
        A, k, B = affine_data(word)
        act = action(word)
        fp = fixed_point(word)
        print()
        print(f"word = {word}")
        print(f"  A = {A}, k = {k}, B = {B}")
        print(f"  2^A = {1 << A}, 3^k = {3 ** k}")
        print(f"  action = {act}")
        print(f"  fixed point = {fp} ≈ {float(fp):.12g}")

        if act == "positive":
            print("  Lock 1 check: positive-action ghost is negative.")
            print(f"  F_w(1) = {F(word, 1)} > 1")
        elif act == "contractive":
            print("  Contractive word: not a Lock 1 positive engine.")
        else:
            print("  Neutral word: should not occur for nonempty words.")


def run_bounded_scan(max_len=6, max_a=6):
    print()
    print("Bounded verification scan")
    print("=========================")
    print(f"Enumerating all words with length <= {max_len}, exponents <= {max_a}")

    total = 0
    positive = 0
    contractive = 0
    neutral = 0

    most_negative = None

    for word in enumerate_words(max_len=max_len, max_a=max_a):
        total += 1
        act = action(word)

        if act == "positive":
            positive += 1
            result = verify_positive_engine(word)
            fp = result["fixed_point"]
            if most_negative is None or fp < most_negative[1]:
                most_negative = (word, fp)

        elif act == "contractive":
            contractive += 1

        else:
            neutral += 1

    print(f"total words      = {total}")
    print(f"positive action  = {positive}")
    print(f"contractive      = {contractive}")
    print(f"neutral          = {neutral}")

    assert neutral == 0, "Nonempty neutral words should not exist."
    print("neutral check    = passed")

    print("Lock 1 scan      = passed")
    print("Every positive-action word scanned had a negative fixed point.")
    print("Every positive-action word scanned increased positive sample x values.")

    if most_negative is not None:
        word, fp = most_negative
        print(f"most negative scanned ghost: word={word}, fixed_point={fp} ≈ {float(fp):.12g}")


if __name__ == "__main__":
    run_examples()
    run_bounded_scan(max_len=6, max_a=6)

Why the Python scan matters
---------------------------

The proof is algebraic and does not depend on the finite scan.

The scan is included so the block formulas, B recurrence, action
classification, and negative-ghost conclusion can be verified directly on many
explicit examples.

The universal proof is the sign argument:

    positive action:
        3^k > 2^A

    nonempty word:
        B_w > 0

    fixed point:
        x = B_w / (2^A - 3^k)

    denominator is negative, numerator is positive, so x < 0.

Therefore no positive-action repeated block has a positive fixed point.
