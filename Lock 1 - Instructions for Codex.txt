MACRO-LOOP / CYCLE LOCK TEST

Goal:
Test the claim that the +1 offset prevents exact nontrivial loop closure.

Do NOT forward-enumerate starting numbers.

Odd-only block:
  F_w(x) = (3^k x + B_w) / 2^A

Cycle condition:
  F_w(x) = x

Therefore:
  D = 2^A - 3^k
  x = B_w / D

Candidate cycle requirements:
  1. D > 0
  2. B_w % D == 0
  3. x is positive odd
  4. x follows the exact exponent word w
  5. all cyclic rotations are valid
  6. exclude trivial word (2) giving x=1

Implement cycle-word scanner.

For each exponent word w:
  compute A, k, B_w incrementally:
    B_{j+1} = 3*B_j + 2^{A_j}
    A_{j+1} = A_j + a_j

  D = 2^A - 3^k

  if D <= 0:
      reject as positive-action/neutral, not positive cycle candidate

  if B_w % D != 0:
      reject by offset divisibility

  x = B_w / D

  if x <= 0 or x even:
      reject

  forward verify:
      run odd-only Collatz from x for k steps
      record observed exponent word
      require observed word == w
      require final value == x

  rotation verify:
      for each cyclic rotation of w:
          compute its candidate x_r
          verify it is one of the cycle values
          verify exact valuation path

Known sanity:
